üéØ 1. System Overview

What is Bote WiFi?

Bote WiFi is an innovative, eco-friendly internet access system that exchanges empty bottles for WiFi time. It uses an IR (Infrared) sensor to detect bottle insertions and automatically grants internet access to users.

üí° Core Concept:

Insert empty bottles ‚Üí System detects them ‚Üí Claim bottles ‚Üí Get internet time

1 Bottle = 5 Minutes of Internet Access

System Components

üì°

Detects bottle insertions automatically
üåê

Access at http://10.0.0.1
üë•

Multiple users can use simultaneously

‚ú® 2. Key Features

  • Automatic Bottle Detection: IR sensor automatically detects when bottles are inserted
  • Multi-User Support: Multiple users can use the system at the same time with individual time tracking
  • Personal Time Balance: Each device has its own time balance that can't be used by others
  • Real-Time Updates: Interface updates every 0.5 seconds to show current status
  • Auto-Claiming: System automatically claims bottles for you when detected
  • Flexible Session Management: Start and stop bottle detection sessions as needed
  • Time Accumulation: Insert multiple bottles to build up internet time
  • Responsive Design: Works on phones, tablets, and computers
‚úÖ Benefits:
  • Encourages recycling and environmental responsibility
  • Fair time allocation - each user gets what they contribute
  • No cash transactions needed
  • Easy to use with minimal setup

For First-Time Users

Click the large üçæ Insert Bottle button on the main screen.

This activates the IR sensor to start detecting bottles.

üí° Automatic Process:

You can insert more bottles at any time to extend your session:

  • No limit on how many bottles you can add
  • Time accumulates continuously

Ending Your Session

To manually end your session and disconnect all users:

‚ö†Ô∏è Important Notes:

üë• 6. Multi-User Functionality

How Multi-User Works

The Bote WiFi system supports multiple users simultaneously. Each device is assigned a unique Device ID that tracks their individual time balance.

üí° Key Concepts:
  • Device ID: Automatically generated unique identifier for your device
  • Personal Time Balance: Each device has its own time counter
  • Shared Bottle Pool: All detected bottles go to a shared "pending" pool
  • First Come, First Served: Users claim bottles from the pending pool

Multi-User Scenario Example

1 Multiple Users Connect

Alice, Bob, and Charlie all connect to Bote WiFi and open the web interface.

Each gets a unique Device ID stored in their browser.

2 Session is Started

Alice clicks "Insert Bottle" button - session becomes active for everyone.

3 Bottles Are Inserted

Someone inserts 5 bottles into the slot.

Pending Bottles counter shows "5" for everyone.

4 Auto-Claiming (Default)

System automatically distributes bottles:

  • Alice's device auto-claims bottles ‚Üí Gets 5 minutes each
  • Bob's device auto-claims bottles ‚Üí Gets 5 minutes each
  • Charlie's device auto-claims bottles ‚Üí Gets 5 minutes each

Note: The system tries to distribute fairly, but auto-claim happens quickly!

5 Individual Time Tracking

Each user sees their own time balance:

  • Alice: 15 minutes (claimed 3 bottles)
  • Bob: 10 minutes (claimed 2 bottles)
  • Charlie: 0 minutes (didn't claim any)

Internet stays active as long as ANY user has time remaining.

Understanding Shared vs Personal

Shared (All Users See) Personal (Your Device Only)
Session Active/Inactive status My Bottles counter
Pending Bottles count My Time Left
Total Users count Device ID
Bottle detection events Individual internet time balance
‚ö†Ô∏è Important Multi-User Rules:
  • You cannot use another user's time - each device has its own balance
  • When all users' time expires, internet disconnects for everyone
  • Ending the session clears ALL users' time balances
  • Pending bottles are available to all users - fastest claims first

Fair Usage Tips

  • Insert bottles you personally want to use for time
  • Don't end the session if other users still have time remaining
  • The system is designed for cooperative use
  • Monitor your own time and add more bottles as needed

‚ùì 10. Frequently Asked Questions

General Questions

Q: How much internet time does one bottle give?

A: Each bottle gives exactly 5 minutes of internet access time.

Q: Can I insert multiple bottles at once?

A: Insert bottles one at a time, waiting 2-3 seconds between each insertion for accurate detection.

Q: Is there a limit to how many bottles I can insert?

A: No limit! You can insert as many bottles as you want and accumulate unlimited time.

Q: What happens if I close my browser?

A: Your time continues counting down even if you close the browser. Your Device ID is saved, so reopening shows your remaining time.

Q: Can I use my accumulated time tomorrow?

A: Time persists as long as the session isn't ended and the server stays running. However, if someone clicks "End Session" or server restarts, all time is cleared.

Multi-User Questions

Q: Can multiple people use the system at the same time?

A: Yes! The system supports unlimited simultaneous users, each with their own time balance.

Q: Can I share my time with a friend?

A: No, each device has its own time balance that cannot be transferred. However, you can insert bottles for your friend to claim.

Q: What happens when all users' time expires?

A: Internet disconnects for everyone when the last user's time reaches 0:00.

Q: How does the system know which user claimed which bottle?

A: Each browser is assigned a unique Device ID. When you claim a bottle, it's associated with your Device ID.

Technical Questions

Q: Do I need to install any app or software?

A: No! Just connect to WiFi and open your web browser. Everything works in the browser.

Q: Why do I need to click "Insert Bottle" button first?

A: This activates the bottle detection system. It's a feature to prevent accidental bottle counting when the system isn't in use.

Q: How accurate is the bottle detection?

A: Very accurate when used properly. Insert bottles slowly, one at a time, and wait for confirmation.

Troubleshooting Questions

Q: The page says "Error connecting to system" - what do I do?

A: This means lost connection to the server. Check your WiFi connection and try refreshing. If problem persists, the server may be down.

Q: Why didn't my bottle get detected?

A: Most common reasons:

  • Session wasn't active (must click Insert Bottle first)
  • Bottle not properly inserted
  • Inserted too quickly after previous bottle
  • Sensor obstruction or malfunction

Q: I have time remaining but no internet access - why?

A: Check that status shows "‚úÖ Internet Active". Try disconnecting and reconnecting to WiFi. If still issues, check with system administrator.

Q: Can I test if the sensor is working?

A: Yes! Watch the session status panel - it should show "‚ú® Bottle Detected!" when sensor detects an object. The system also has a test endpoint for admins.

Usage Questions

Q: What types of bottles can I use?

A: Any bottle that fits in the slot and can break the IR sensor beam. Typically plastic bottles or similar-sized containers.

Q: Do bottles need to be empty?

A: Yes, for safety and hygiene reasons. The system encourages recycling of empty bottles.

Q: How long does it take for time to be added after inserting a bottle?

A: Nearly instant! Detection ‚Üí Auto-claim ‚Üí Time added happens within 1-2 seconds.

Q: Can I add more bottles while using internet?

A: Absolutely! You can insert more bottles at any time to extend your session.

Account & Privacy Questions

Q: Do I need to create an account?

A: No account needed! System automatically creates a Device ID for your browser.

Q: Is my browsing activity tracked?

A: The system only tracks bottles claimed and time remaining. No browsing activity is monitored or logged.

Q: What happens if I clear my browser data?

A: Your Device ID is stored in localStorage. Clearing browser data will generate a new ID and you'll lose any accumulated time.

Q: Can I use the same time balance on multiple devices?

A: No, each device/browser combination has its own separate time balance.
